# v0.2.0-to-v0.3.0 migration guide

이 문서는 Hyper-Waterfall을 이미 적용한 저장소가 `v0.2.0` baseline에서 다국어 적용 구조가 포함된 `v0.3.0`으로 이동할 때 확인해야 할 항목을 정리한다. `v0.3.0` GitHub Release/tag와 npm publish는 완료됐으며, 실제 업데이트는 목표 release/tag의 `templates/manifest.json`, 적용 저장소의 `.hyper-waterfall/version.json`, 이 migration guide를 함께 확인한 뒤 승인된 일반 task 흐름으로 수행한다. Homebrew/Docker/plugin 배포는 별도 채널 작업으로 분리한다.

## 대상 버전

| 항목 | 값 |
|---|---|
| 시작 version | `v0.2.0` |
| 목표 version | `v0.3.0` |
| canonical 배포 단위 | GitHub Release/tag |
| manifest 위치 | `templates/manifest.json` |
| release 상태 | `released` |
| 적용 저장소 version/locale 기록 | `.hyper-waterfall/version.json` |
| 선택 locale 기록 필드 | top-level `locale` |
| 기본 locale | `en` |
| 초기 지원 locale | `en`, `ko`, `zh-CN` |
| fallback locale | `en` |
| manifest locale 대상 | 15 entries |
| manifest update policy 요약 | `merge=4`, `overwrite=11`, `preserve=8`, `symlink=2` |

주의: `v0.3.0` release 기준의 `package.json`과 `templates/manifest.json`은 `0.3.0`, `v0.3.0`, `release.status: released`로 정렬된다. Homebrew tap 갱신과 Docker/plugin 배포는 이 migration guide만으로 실행하지 않고 별도 채널 작업과 작업지시자 승인을 거친다.

## 변경 요약

- README 진입 문서를 `README.md` 기본 영어, `README.ko.md` 한국어, `README.zh-CN.md` 중국어 간체로 분리한다.
- `docs/localization.md`에 다국어 적용 정책을 정리하고, `en`, `ko`, `zh-CN`을 초기 지원 locale로 둔다.
- `templates/locales/{en,ko,zh-CN}/...` mirror 구조를 추가해 적용 저장소의 AGENTS, CLAUDE, GitHub Issue/PR template, `mydocs/_templates`, manual, Skill, 폴더 README를 선택 locale 기준으로 제공한다.
- `templates/manifest.json`에 top-level `localization` 계약과 entry-level `files[].localization` 계약을 둔다.
- 신규 적용 시 승인된 locale은 `.hyper-waterfall/version.json`의 top-level `locale`에 기록한다.
- 기존 적용 저장소 update는 `.hyper-waterfall/version.json`의 `locale`을 기본으로 보존한다. 기록이 없으면 `unknown`으로 보고하고 보존 또는 전환 승인을 받는다.
- `docs/lifecycle/adoption.md`, `docs/lifecycle/update.md`, `docs/agent-entrypoint.md`가 선택 locale, fallback 후보, locale manifest diff, locale 보존/전환 판단을 파일 변경 전 판단 보고에 포함하도록 정렬된다.
- CLI `init`/`update` dry-run은 선택 locale, unsupported locale, 기존 locale 보존, 명시적 locale 전환 요청을 판단 결과로 출력한다.

## 추가 파일

| 경로 | 목적 | 적용 저장소 처리 |
|---|---|---|
| `README.ko.md` | framework 한국어 README | 적용 저장소에 복사하지 않는다. framework 사용자 진입 문서다. |
| `README.zh-CN.md` | framework 중국어 간체 README | 적용 저장소에 복사하지 않는다. framework 사용자 진입 문서다. |
| `docs/localization.md` | 다국어 적용 정책 | 적용 저장소에 복사하지 않는다. update 판단과 review 근거로 참조한다. |
| `docs/localization-smoke.md` | locale별 smoke 검증 기준과 결과 | 적용 저장소에 복사하지 않는다. release 검증 근거로 참조한다. |
| `docs/migrations/v0.2.0-to-v0.3.0.md` | 이번 version 이동의 수동 확인 항목 | Hyper-Waterfall 버전 업데이트 PR 본문과 review 근거로 사용한다. |
| `templates/locales/en/**` | 영어 locale pack | 선택 locale이 `en`이면 manifest의 `sourcePattern` 후보로 사용한다. |
| `templates/locales/ko/**` | 한국어 locale pack | 기존 한국어-only 적용 저장소를 보존하려면 maintainer 승인 후 `ko` 후보로 사용한다. |
| `templates/locales/zh-CN/**` | 중국어 간체 locale pack | 명시 전환 요청이 있을 때 별도 승인 후 후보로 사용한다. |

| `docs/lifecycle/update.md` | 기존 적용 저장소 update 판단 결과에 현재 locale, 요청 locale, 전환 요청, locale manifest diff를 포함 | framework 문서. update 판단 시 참조 |
| `docs/lifecycle/update_pr.md` | Hyper-Waterfall 버전 업데이트 PR에 locale 판단 결과를 반영 | framework 문서. update PR 작성 시 참조 |

| `templates/.github/ISSUE_TEMPLATE/task.yml`, `templates/.github/pull_request_template.md` | 선택 locale pack 대상이 되도록 manifest 계약과 mirror source 정렬 | `merge` 후보. 저장소별 GitHub 운영 방식과 충돌 가능 |
| `templates/mydocs/_templates/**` | 수행계획서, 구현계획서, 단계 보고서, 최종 보고서 등의 작성 언어를 선택 locale 기준으로 정렬 | `overwrite` 후보. 생성 문서 형식 변경이므로 기존 수정 여부 확인 |
| `templates/mydocs/manual/**` | lifecycle, document structure, release/update protocol, task workflow에 locale 판단 반영 | `overwrite` 후보. `requiresSemanticReview: true` |
| `templates/mydocs/skills/**` | task-start, task-stage-report, task-final-report 등 Skill 본문을 locale pack mirror로 제공 | `overwrite` 후보. `requiresSemanticReview: true` |
| `templates/mydocs/*/README.md` | 각 산출물 폴더 README를 선택 locale 기준으로 제공 | `overwrite` 후보 |
| `bin/hyper-waterfall.js`, `test/cli-smoke.test.js` | CLI dry-run과 smoke test에 locale 판단을 연결 | framework 코드. 적용 저장소 대상 아님 |

## 적용 저장소 수동 확인

기존 적용 저장소를 `v0.3.0`으로 올릴 때 maintainer는 먼저 `.hyper-waterfall/version.json`을 확인한다.

| 현재 상태 | update 판단 | 기본 처리 |
|---|---|---|
| `locale` 필드 없음 | 현재 locale을 `unknown`으로 보고 | 자동 추론하지 않는다. 한국어-only 문맥을 유지하려면 `ko` 선택을 승인받는다. |
| `locale: ko` | 기존 `ko`를 기본 보존 | `selectedForDiff: ko`로 locale manifest diff를 검토한다. |
| `locale: en` | 기존 `en`을 기본 보존 | 영어 pack 기준으로 diff를 검토한다. |
| `locale: zh-CN` | 기존 `zh-CN`을 기본 보존 | 중국어 간체 pack 기준으로 diff를 검토한다. |
| unsupported locale | 선택 locale source 누락과 fallback 후보를 보고 | 조용히 영어로 대체하지 않고 수동 승인 후 처리한다. |
| `--locale en` 또는 `--locale zh-CN` 명시 | locale 전환 요청으로 보고 | 기존 locale 보존의 부수 효과가 아니라 별도 승인 항목으로 다룬다. |

수동 확인 체크리스트:

- `.hyper-waterfall/version.json`의 `frameworkVersion`, `releaseTag`, `locale`, `updatedAt`을 확인한다. 기존 기록은 `preserve` 대상이므로 임의로 덮어쓰지 않는다.
- 한국어-only v0.2.0 적용 저장소는 `locale`이 없을 수 있다. 이 경우 CLI가 `unknown`을 보고하면 maintainer가 `ko` 보존 또는 다른 locale 전환을 명시 승인해야 한다.
- `AGENTS.md`와 `CLAUDE.md`는 프로젝트별 운영 규칙을 담을 수 있으므로 `merge` 후보로 보고 수동 diff를 확인한다.
- `.github/ISSUE_TEMPLATE/task.yml`과 `.github/pull_request_template.md`는 저장소별 운영 방식에 맞게 수정됐을 수 있으므로 자동 overwrite하지 않는다.
- `mydocs/_templates/`는 생성 문서의 섹션명과 작성 언어를 바꾸므로 기존 local customization이 있는지 확인한다.
- `mydocs/manual/`과 `mydocs/skills/`는 절차 계약을 담는다. locale mirror로 바꾸더라도 승인 게이트, Issue 기준 추적, stage report, final report, PR 게시 절차의 의미가 유지되는지 semantic review를 수행한다.
- `.agents/skills`와 `.claude/skills`는 symlink 대상이다. 기존 파일이나 directory가 있으면 symlink로 바꾸기 전에 승인받는다.
- `docs/`, `src/`, `examples/`, `schemas/`, `package.json`, `tsconfig.json`은 adoption/update 대상 파일이 아니다. 프로젝트별 공식 제품 문서는 별도 task의 문서 위치 판단을 거쳐야 한다.

## 충돌 가능성

| 영역 | 충돌 이유 | 기본 처리 |
|---|---|---|
| `locale` 미기록 version state | v0.2.0 적용 저장소는 선택 locale 기록이 없을 수 있음 | `unknown`으로 보고하고 maintainer가 `ko` 보존 또는 전환을 승인 |
| language mixing | 일부 파일만 새 locale pack으로 바뀌면 AGENTS, template, manual, Skill 언어가 섞일 수 있음 | locale manifest diff와 수동 확인 항목에 혼합 위험 기록 |
| `AGENTS.md`, `CLAUDE.md` | 대상 저장소의 로컬 운영 규칙이 들어 있을 수 있음 | `merge`, 수동 review |
| `.github/` | 이슈/PR template은 저장소 운영 관습에 맞게 수정될 수 있음 | `merge`, 수동 review |
| `mydocs/_templates/` | 생성 문서 섹션명과 작성 언어가 바뀜 | 기존 사용자 수정 여부 확인 후 승인 |
| `mydocs/manual/`, `mydocs/skills/` | 절차 의미가 바뀌면 하이퍼-워터폴 계약이 깨짐 | `requiresSemanticReview: true`, semantic review 필수 |
| unsupported locale | 선택 locale source가 없을 수 있음 | 누락과 fallback 후보를 보고하고 자동 적용 금지 |
| `.agents/skills`, `.claude/skills` | 기존 경로가 파일/폴더이면 symlink와 충돌 | 기존 경로 확인 후 승인 필요 |
| `.hyper-waterfall/version.json` | 상태 파일이므로 기존 기록 보존 필요 | `preserve`, 누락 필드 추가도 승인 후 수행 |

## 검증

프레임워크 저장소에서 이번 migration guide 자체는 다음 명령으로 검증한다.

```bash
test -f docs/migrations/v0.2.0-to-v0.3.0.md
grep -nE '대상 버전|변경 요약|추가 파일|수정 파일|수동 확인|충돌 가능성|검증|후속 작업' docs/migrations/v0.2.0-to-v0.3.0.md
grep -nE 'v0.2.0|v0.3.0|manifest|version|locale|\.hyper-waterfall' docs/migrations/v0.2.0-to-v0.3.0.md
git diff --check
```

적용 저장소 Hyper-Waterfall 버전 업데이트 PR에서는 다음 확인을 추가한다.

- 현재 `.hyper-waterfall/version.json`의 `frameworkVersion`, `releaseTag`, `locale`을 확인한다.
- `locale`이 없으면 `unknown`으로 보고하고, 한국어 문맥 보존 시 `ko` 선택 승인을 받는다.
- `templates/manifest.json` 기준 `supportedLocales`, `fallbackLocale`, `preserveSelectedLocaleOnUpdate`, localized entry source 존재 여부를 확인한다.
- `hyper-waterfall update --dry-run` 또는 동등한 판단 출력에서 current locale, requested locale, target release locale support, migration guide, manifest diff, locale manifest diff, locale 보존/전환 판단, 승인 요청이 보이는지 확인한다.
- `mydocs/manual/`과 `mydocs/skills/`의 locale mirror는 literal 번역 여부가 아니라 절차 의미 보존을 확인한다.
- PR 본문에는 자동 적용 가능, 수동 확인 필요, conflict, 보류 항목을 분리해 남긴다.

## Hyper-Waterfall 버전 업데이트 PR 본문 반영 기준

이 migration guide를 근거로 Hyper-Waterfall 버전 업데이트 PR을 만들 때는 기존 PR 템플릿을 사용하되, 다음 항목을 반드시 남긴다.

| PR 본문 항목 | 이번 migration에서 채울 내용 |
|---|---|
| 현재 version | `v0.2.0` |
| 목표 release/tag | `v0.3.0` |
| 현재 locale | `.hyper-waterfall/version.json`의 `locale`; 없으면 `unknown` |
| 요청 locale 또는 전환 요청 | 명시 요청이 없으면 기존 locale 보존, 명시 요청이 있으면 별도 승인 항목 |
| migration guide | `docs/migrations/v0.2.0-to-v0.3.0.md` |
| manifest diff | localized entry 15개, 비대상 entry 10개, `merge=4`, `overwrite=11`, `preserve=8`, `symlink=2` 요약 |
| locale manifest diff | 선택 locale source 존재, fallback source 존재, missing source 여부 |
| 자동 적용 가능 | checksum이 `ready`이고 사용자 수정이 없는 file 후보. directory/root checksum은 `pending-release`이면 수동 확인 |
| 수동 확인 필요 | `AGENTS.md`, `CLAUDE.md`, `.github/`, `mydocs/_templates/`, `mydocs/manual/`, `mydocs/skills/`, `.hyper-waterfall/version.json` |
| conflict | 사용자 수정 파일과 목표 release 변경이 같은 경로를 수정하는 경우, unsupported locale, symlink 경로 충돌, language mixing |
| 보류 | Homebrew/Docker/plugin 배포, write mode 자동 적용, 새 locale 추가 |

## CLI 출력 기준

`v0.2.0 -> v0.3.0` update/doctor 출력은 이 문서를 근거로 다음 항목을 보여줘야 한다. 실제 파일 변경이나 PR 생성은 이 출력만으로 자동 수행하지 않고, 작업지시자 승인 후 일반 task 흐름으로 전환한다.

| 출력 항목 | 이번 migration 기준 |
|---|---|
| 현재 version | `.hyper-waterfall/version.json`의 `frameworkVersion` 또는 `releaseTag` |
| 현재 locale | `.hyper-waterfall/version.json`의 top-level `locale`; 없으면 `unknown` |
| 요청 locale | CLI 인자로 명시된 `--locale`; 없으면 none |
| 목표 release/tag | `v0.3.0` 또는 사용자가 지정한 목표 release/tag |
| 목표 release locale 지원 | `en`, `ko`, `zh-CN`, fallback `en`, availability `complete` |
| migration guide | `docs/migrations/v0.2.0-to-v0.3.0.md` |
| manifest diff | manifest total, update policy, target status 요약 |
| locale manifest diff | 선택 locale source와 fallback source의 exists/missing 상태 |
| locale 보존/전환 판단 | 기존 locale 보존, `unknown` 처리, 또는 `ko -> zh-CN` 같은 전환 요청 |
| 검증 | dry-run, locale source 존재, semantic review, PR 본문 수동 확인 |
| 승인 요청 | Hyper-Waterfall 버전 업데이트 PR 생성 여부와 locale 보존/전환 범위 |

## 후속 작업

- Homebrew/Docker/plugin 배포는 이 migration guide로 수행하지 않는다. 별도 채널 task와 작업지시자 승인을 거친다.
- release task에서는 `templates/manifest.json`의 version metadata, checksum 상태, migration guide 링크를 목표 release 상태에 맞게 확인했다.
- 실제 write mode 적용은 이번 smoke 범위가 아니다. CLI dry-run 판단을 승인한 뒤 일반 Hyper-Waterfall task 흐름으로 파일 변경을 수행한다.
- 새 locale 추가, 번역 품질 개선, manual/Skill semantic review 자동화는 별도 이슈로 분리한다.
