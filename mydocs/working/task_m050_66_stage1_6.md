## 본문 변경 정도 / 본문 무손실 여부

`README.ko.md`는 대폭 재작성했다. 다만 사용자가 보존을 요청한 핵심 정보는 유지했다.

유지·강화한 내용:

- 상단 후킹 요약에 `한 줄 적용`, `작업 산출물 구조화`, `공식 프롬프팅 가이드 정합`, `Multi-agent 운영`을 포함했다.
- `바로 설치`를 상단 후킹 요약 바로 아래로 이동했다.
- `이 저장소가 한 일`의 세 하위 섹션을 보존했다.
- `도입 후 작업 흐름`, `타스크 진행 절차`, `핵심 SKILL 상세`, `타스크 사이클`, `문서 구조`를 유지했다.
- 문서 구조 정책 문단, `manual/`, `tech/`, `_templates/`, GitHub Issue/PR 경계 설명을 유지했다.
- 방법론 설명, 핵심 구조, 핵심 원칙, 역할 분담, 바이브 코딩 비교, 왜 강력한가, 살아있는 예시, 설계 원칙, 프롬프트 가이드 준수는 `부록` 아래로 묶었다.

정리한 내용:

- `새 저장소에 빠르게 적용하기` heading은 `바로 설치`와 중복되므로 남기지 않았다.
- 시간에 취약한 특정 모델명 중심 표현은 사용하지 않았다.
- CLI, Homebrew, update lifecycle 설명은 `유지보수자 세부 정보`의 접힘 영역으로 이동했다.


결과:

- OK: 후킹 요약, `바로 설치`, `이 저장소가 한 일`, 운영 흐름, 문서 구조, 부록, 프롬프트 가이드 준수가 확인됐다.
- OK: heading 순서가 `바로 설치 -> 적용하면 작업 방식이 이렇게 바뀝니다 -> 왜 이 저장소를 써야 하는가 -> 이 저장소가 한 일 -> 도입 후 작업 흐름 -> 적용 후 대상 저장소 구조 -> 유지보수자 세부 정보 -> 부록 -> 라이선스`로 확인됐다.
- OK: 문서 구조 정책 문구가 유지됐다.
- OK: `새 저장소에 빠르게 적용하기`, `OpenAI GPT`, `Claude Opus` 검색은 결과 없음(exit 1)으로 확인됐다.
- OK: `git diff --check`가 빈 출력으로 통과했다.
