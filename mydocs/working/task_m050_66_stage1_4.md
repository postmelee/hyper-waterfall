## 단계 목적

현재 부록으로 이동한 방법론 설명은 유지하면서, upstream README의 `postmelee/hyper-waterfall` 블록 중 `이 저장소가 한 일` 내용을 상단에 최대한 그대로 복원했다. 단, 사용자가 요청한 대로 특정 원본 저장소에 종속적으로 보이는 표현은 일반 하네스 관점으로 바꿨고, `새 저장소에 빠르게 적용하기`는 상단 `빠른 시작`과 중복되므로 다시 추가하지 않았다.

## 본문 변경 정도 / 본문 무손실 여부

`README.ko.md`의 상단 흐름과 부록은 유지했다.

유지한 흐름:

- `빠른 시작`
- `언제 쓰면 좋은가`
- `왜 이 저장소를 써야 하는가`
- `이 하네스가 하는 일`
- `무엇이 바뀌나`
- `도입 후 작업 흐름`
- `부록`

upstream `이 저장소가 한 일`에서 복원한 내용:

- `프롬프트 한 줄로 어떤 저장소에든 적용 — 모듈화 + placeholder 치환`
- `공식 프롬프팅 가이드와의 정합`
- `Multi-agent 호환 + 운영 규칙·SKILL·매뉴얼 분리로 토큰·컨텍스트 효율`
- 토큰 효율, 컨텍스트 효율, 의도 전달 명확성, 모델 간 이식성 효과 목록

일반화한 표현:

- `이 저장소가 한 일` heading은 `이 하네스가 하는 일`로 변경했다.
- `원본 방법론(rhwp)`과 `rhwp는 ...` 비교 문장은 특정 원본 저장소에 기대지 않는 일반 설명으로 바꿨다.
- OpenAI/Anthropic 링크와 모델명 중심 표현은 상단에서 반복하지 않고, 부록의 `프롬프트 가이드 준수`로 연결했다.

## 검증 결과

실행 명령:

```bash
rg -n "이 하네스가 하는 일|프롬프트 한 줄로 어떤 저장소에든 적용|공식 프롬프팅 가이드와의 정합|Multi-agent 호환|토큰 효율|컨텍스트 효율|의도 전달 명확성|모델 간 이식성" README.ko.md
rg -n "새 저장소에 빠르게 적용하기|rhwp에서 처음 도입|다음과 같이 확장|원본 방법론\\(rhwp\\)|rhwp는" README.ko.md
rg -n "부록|Hyper-Waterfall이란\\?|핵심 구조|핵심 원칙|바이브 코딩 vs Hyper-Waterfall|프롬프트 가이드 준수" README.ko.md
git diff --check
```

결과:

- OK: `이 하네스가 하는 일`과 세 하위 섹션, 효과 목록이 확인됐다.
- OK: `새 저장소에 빠르게 적용하기`, `rhwp에서 처음 도입`, `다음과 같이 확장`, `원본 방법론(rhwp)`, `rhwp는` 검색은 결과 없음(exit 1)으로 확인됐다.
- OK: 부록 heading과 `프롬프트 가이드 준수` heading이 유지됐다.
- OK: `git diff --check`가 빈 출력으로 통과했다.
