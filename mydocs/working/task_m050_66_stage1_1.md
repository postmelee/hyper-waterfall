## 본문 변경 정도 / 본문 무손실 여부

`README.ko.md`는 Stage 1의 적용자 중심 구조를 유지하면서 본문을 보강했다. `무엇이 바뀌나`의 기존 비교 표는 유지했고, 그 아래에 원본의 `적용하면 바로 달라지는 것` 10개 항목을 짧은 설명 중심으로 복원했다.

원본 README의 다음 메시지는 새 한국어 기준본에 다시 반영됐다.

- 거시적 워터폴의 규율과 미시적 애자일의 속도를 AI가 동시에 가능하게 한다.
- 사람은 절대 생각을 멈추지 않는다.
- 작업 이력이 다음 작업의 프롬프트가 된다.
- 방향 오류는 큰 구현 이후가 아니라 계획서와 Stage 경계에서 발견한다.
- AI는 결정자가 아니라 배율기이며, 사람은 결정에 AI는 실행에 집중한다.
- 세션은 짧게 유지하고 기억은 저장소에 남긴다.

## 검증 결과

실행 명령:

```bash
rg -n "왜 이 저장소를 써야 하는가|작업 이력이 다음 작업의 프롬프트|AI가 잘할 수밖에|적용하면 바로 달라지는 것|사람은 절대 생각을 멈추지 않는다|같은 규율, 100배 빠른 속도|AI는 배율기" README.ko.md
rg -n "English|한국어|简体中文|빠른 시작|언제 쓰면 좋은가|왜 이 저장소를 써야 하는가|무엇이 바뀌나|적용 후 구조|유지보수자|부록" README.ko.md
git diff --check
```

결과:

- OK: 설득력 보강 핵심 문구가 `README.ko.md`에 존재함을 확인했다.
- OK: 언어 링크, 빠른 시작, 언제 쓰면 좋은가, 왜 이 저장소를 써야 하는가, 무엇이 바뀌나, 적용 후 구조, 유지보수자, 부록 heading을 확인했다.
- OK: `git diff --check`가 빈 출력으로 통과했다.

수동 확인:

- OK: `무엇이 바뀌나` 표는 유지됐다.
- OK: 10개 체감 효과는 원본 의미를 유지하되 각 항목을 한 줄 또는 짧은 보충 설명으로 축약했다.
- OK: `왜 이 저장소를 써야 하는가`는 지식 자산화, 방향 오류 조기 발견, 자동화된 역할 분담, 세션 경량화로 도입 이유를 설명한다.
- OK: 부록은 원저자 방법론 표현과 핵심 원칙을 Stage 1보다 더 충실히 보존한다.
