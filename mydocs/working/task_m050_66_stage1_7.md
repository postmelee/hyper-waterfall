## 본문 변경 정도 / 본문 무손실 여부

`README.ko.md`는 상단 후킹 표와 상세 구현 섹션 제목만 좁게 수정했다. `구현 확장` 섹션 아래의 세부 본문은 삭제하지 않았고, 기존 모듈화·placeholder 치환·공식 프롬프팅 가이드 정합·Multi-agent 호환 설명도 유지했다.

변경한 관점:

- `이 저장소는 ... 패키지화합니다`에서 `이 하네스를 적용하면 ... 운영 레일이 설치됩니다`로 변경했다.
- 후킹 표는 `한 줄 적용`, `승인 게이트`, `작업 기억 외부화`, `공식 프롬프팅 가이드 정합`, `Multi-agent 운영`으로 재구성했다.
- `이 저장소가 한 일` heading은 `구현 확장: 방법론을 하네스로 만든 방식`으로 변경했다.

## 검증 결과

실행 명령:

```bash
rg -n "이 하네스를 적용하면|제공되는 것|승인 게이트|작업 기억 외부화|공식 프롬프팅 가이드 정합|Multi-agent 운영" README.ko.md
rg -n "구현 확장: 방법론을 하네스로 만든 방식" README.ko.md
rg -n "^## 이 저장소가 한 일|#이-저장소가-한-일|\\[이 저장소가 한 일\\]" README.ko.md
git diff --check
```

결과:

- OK: 상단 후킹 문구와 표 항목이 확인됐다.
- OK: `## 구현 확장: 방법론을 하네스로 만든 방식` heading이 확인됐다.
- OK: 기존 `## 이 저장소가 한 일` heading과 관련 앵커 참조 검색은 결과 없음(exit 1)으로 확인됐다.
- OK: `git diff --check`가 빈 출력으로 통과했다.
