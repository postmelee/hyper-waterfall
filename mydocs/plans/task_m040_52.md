# Task #52 수행계획서 - Codex plugin public 배포와 UI discovery smoke

GitHub Issue: [#52](https://github.com/postmelee/hyper-waterfall/issues/52)
마일스톤: M040

## 목적

Task #38에서 생성한 Codex plugin repo-local 배포 후보가 Codex Plugin Directory 또는 CLI `/plugins` 진입면에서 실제로 발견 가능한지 확인한다.

또한 public 배포 전에 필요한 install-surface metadata, asset, screenshot, legal link 조건을 재점검하고, 작업지시자 별도 승인에 따라 public 배포를 실행하거나 보류 사유와 대체 설치 경로를 문서화한다.

## 배경

Task #38은 `plugins/hyper-waterfall-codex/`와 `.agents/plugins/marketplace.json`을 만들고 `codex plugin marketplace add .` local smoke를 통과했다. 다만 Codex restart 이후 Plugin Directory UI 표시와 thin wrapper Skill invocation은 수동 확인 또는 interactive 환경 확인이 필요해 자동 검증 범위 밖으로 남았다.

Public 배포는 외부 공개 action이므로 하이퍼-워터폴 승인 게이트와 plugin 후보 품질 점검을 먼저 거쳐야 한다. 이번 task는 #38의 잔여 검증과 public 배포 의사결정을 하나의 추적 가능한 산출물로 정리한다.

참고 산출물:

- `plugins/hyper-waterfall-codex/`
- `.agents/plugins/marketplace.json`
- `mydocs/report/task_m040_38_report.md`
- `mydocs/tech/task_m040_38_codex_plugin_smoke.md`
- OpenAI Codex plugin 공식 문서

## 범위

### 포함

- Codex plugin 공식 문서와 현재 Codex CLI/plugin surface 재확인
- `plugins/hyper-waterfall-codex/` manifest, README, thin wrapper Skill의 public 배포 적합성 점검
- Codex Plugin Directory UI 또는 CLI `/plugins`에서 `hyper-waterfall` 후보 discovery 확인
- thin wrapper Skill invocation smoke 또는 수동 확인 결과 기록
- public 배포용 metadata, asset, screenshot, privacy/terms URL 필요 여부 검토
- 작업지시자 별도 승인 이후 public 배포 실행 또는 보류 사유 문서화
- smoke와 배포 과정에서 생기는 local config/cache 부산물 cleanup 기록

### 제외

- Codex plugin hook guardrail 구현
- Claude plugin packaging 또는 public 배포
- core Skill 본문을 Codex plugin 전용으로 fork하는 작업
- npm publish, Homebrew, Docker 변경
- 작업지시자 승인 없는 public 배포 실행
- PR merge, issue close, release 생성

## 설계 방향

- #38의 hook 없는 thin wrapper bundle 원칙을 유지한다.
- `templates/manifest.json`, migration guide, manual 전문, core Skill 원문은 plugin bundle에 복제하지 않는다.
- public 배포 전에 manifest metadata와 install-surface 표현을 먼저 검토하고, 부족한 항목은 최소 수정 또는 보류 사유로 분리한다.
- Codex Plugin Directory UI 확인은 가능한 경우 Codex 앱/CLI interactive surface로 수행하고, 자동화가 제한되면 확인 한계와 재현 절차를 문서화한다.
- public 배포 실행은 수행계획서 승인과 별도로 작업지시자의 명시 승인이 있는 경우에만 수행한다.

## 리스크

- **Codex UI 자동화 한계**: Plugin Directory UI 확인은 Codex restart 또는 interactive surface가 필요할 수 있다. 자동 확인이 불가능하면 수동 재현 절차와 확인 한계를 문서화한다.
- **공식 문서 변경 가능성**: Codex plugin public distribution 요건이 #38 이후 바뀌었을 수 있다. Stage 1에서 최신 공식 문서를 먼저 확인한다.
- **Public 배포 승인 게이트**: public 배포는 외부 공개 action이다. 별도 명시 승인이 없으면 실행하지 않고 보류 사유를 남긴다.
- **Local config/cache 부산물**: marketplace add/remove나 UI smoke가 사용자 local 설정을 바꿀 수 있다. 실행 전 범위를 기록하고 cleanup 결과를 확인한다.
- **Thin wrapper discoverability 부족**: wrapper Skill만으로 discovery가 부족하면 release snapshot bundle 또는 metadata 보강을 후속 후보로 분리한다.
