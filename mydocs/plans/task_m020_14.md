# task_m020_14.md - README에 관련 문서 링크와 탐색 안내 보강 수행계획서

GitHub Issue: [#14](https://github.com/postmelee/hyper-waterfall/issues/14)
마일스톤: M020

## 목적

`origin/main`의 최신 README를 기준으로, README 독자가 관련 세부 문서로 자연스럽게 이동할 수 있게 탐색 링크와 짧은 안내 문구를 보강한다. README는 개요와 길잡이 역할에 집중하고, 폴더별 규칙·타스크 절차·Git/PR 운용 같은 세부 내용은 기존 매뉴얼의 해당 섹션으로 연결한다.

## 배경

작업지시자는 README의 `문서 구조` 목차나 설명에서 문서 구조 매뉴얼의 `폴더별 상세 규칙` 섹션을 링크하면 독자가 저장소를 이해하고 탐색하는 데 도움이 될 수 있다고 제안했다. 검토 결과, README 안에서는 `https://github.com/.../blob/main/...` 형태보다 저장소 상대 링크가 유지보수와 브랜치 탐색에 적합하다.

현재 로컬 `main`은 `origin/main`보다 1커밋 뒤처져 있고, `origin/main`의 최신 README에는 Hyper-Waterfall 개요와 적용 효과가 확장되어 있다. 메인 worktree에는 작업지시자가 만든 것으로 보이는 미추적 파일 `DESIGN.md`, `README_draft.md`, `opus_report.md`가 있으므로 이 task는 `/private/tmp/hyper-waterfall-task14` 별도 worktree에서 진행한다.

## 범위

### 포함

- `README.md`를 `origin/main` 최신 내용 기준으로 보강
- `도입 후 작업 흐름`, `핵심 SKILL 상세`, `문서 구조`, `적용 후 대상 저장소 구조` 주변의 관련 매뉴얼 링크 추가
- 링크 대상 후보:
  - `templates/mydocs/manual/task_workflow_guide.md`
  - `templates/mydocs/manual/document_structure_guide.md`
  - `templates/mydocs/manual/git_workflow_guide.md`
  - `templates/mydocs/manual/pr_process_guide.md`
- README 내 저장소 문서 링크는 상대 링크와 heading fragment를 우선 사용

### 제외

- 매뉴얼 본문 변경
- SKILL 본문 변경
- README 전체 구조 재작성 또는 대규모 문구 개편
- 새 label 또는 milestone 생성
- 메인 worktree의 미추적 파일 수정, 이동, 삭제

## 설계 방향

- README는 요약과 탐색 허브 역할을 맡기고, 세부 규칙은 기존 매뉴얼로 연결한다.
- 링크는 해당 설명 바로 아래에 짧은 `자세한 규칙은 ...` 문장 또는 compact bullet로 배치한다.
- 문서 구조 예시는 유지하되, 폴더별 상세 규칙·중앙 템플릿 정책·Agent Skills 위치 정책 같은 세부 heading으로 바로 들어갈 수 있게 한다.
- `blob/main` 고정 URL은 README 본문에는 추가하지 않는다. README가 저장소 안에서 렌더링될 때 상대 링크가 더 안정적이며, PR 본문처럼 브랜치 삭제 이후 링크 보존이 중요한 맥락이 아니기 때문이다.
- 매뉴얼과 SKILL 본문을 바꾸지 않으므로 AGENTS.md의 "SKILL이나 매뉴얼 본문 변경 시 README/task_workflow_guide 동기화" 강제 규칙에는 해당하지 않는다.

## 잠정 단계

- **Stage 1 — 작업 흐름과 SKILL 탐색 링크 보강**
  - `도입 후 작업 흐름`, `타스크 진행 절차`, `핵심 SKILL 상세` 주변에 `task_workflow_guide.md`, `pr_process_guide.md`, `git_workflow_guide.md`로 이어지는 안내 링크를 추가한다.
  - 검증 관점: README가 절차 요약과 세부 매뉴얼의 역할을 중복 없이 연결하는지 확인한다.
- **Stage 2 — 문서 구조와 대상 저장소 구조 링크 보강**
  - `문서 구조`, `적용 후 대상 저장소 구조` 주변에 `document_structure_guide.md`의 `폴더별 상세 규칙`, `중앙 템플릿 정책`, `Agent Skills 위치 정책` 링크를 추가한다.
  - 검증 관점: 작업지시자가 제안한 문서 구조 상세 링크 취지가 반영되고, 상대 링크 fragment가 실제 heading과 맞는지 확인한다.
- **Stage 3 — README 링크 품질 검증과 문구 정리**
  - README 전체에서 링크 문구, 상대 링크 사용, `blob/main` 불필요 추가 여부, 기존 설명과 매뉴얼 정합성을 점검한다.
  - 검증 관점: `git diff --check`, 관련 링크 grep, 상태 확인을 통과한다.

- 메인 worktree의 기존 미추적 파일 `DESIGN.md`, `README_draft.md`, `opus_report.md`를 수정하지 않았음을 확인한다.

## 리스크

- **README 과밀화**: 링크를 너무 많이 추가하면 README가 매뉴얼 색인처럼 보일 수 있다. 설명 흐름이 끊기지 않도록 섹션별 핵심 링크만 배치한다.
- **상대 링크 fragment 불일치**: 한국어 heading fragment가 렌더러별로 다르게 처리될 수 있다. GitHub Markdown 기준의 heading 텍스트와 링크 fragment를 맞추고, 가능한 기존 heading에 연결한다.
- **동시 작업 충돌**: `local/task13` 등 다른 README 작업 브랜치가 있다. 이번 task는 `origin/main` 기준 별도 worktree에서 좁게 수정하고, PR 전 최신 base와 충돌 여부를 확인한다.
