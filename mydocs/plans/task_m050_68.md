# #68 영어 locale pack 작성과 한국어 원문 보존 수행계획서

GitHub Issue: [#68](https://github.com/postmelee/hyper-waterfall/issues/68)
마일스톤: M050

## 목적

이번 task의 목적은 Hyper-Waterfall 하네스를 새 저장소에 적용할 때 기본 언어로 사용할 수 있는 영어 locale pack을 만들고, 기존 한국어 원문을 `ko` locale pack의 기준 자료로 보존하는 것이다.

영어 pack은 단순 번역 파일 묶음이 아니라 적용 저장소의 에이전트 운영 규칙, GitHub Issue/PR template, `mydocs/` 생성 문서 template, 폴더 안내 문서가 선택 locale과 일관되게 보이도록 하는 사용자-facing 기본 pack이어야 한다. 동시에 승인 게이트, GitHub Issue 기준 추적, stage 보고, PR 게시, placeholder 보존 같은 Hyper-Waterfall 절차 계약은 한국어 원문과 같은 의미로 유지한다.

## 배경

M050 정책은 기본 locale을 `en`, 기존 한국어 보존 locale을 `ko`, 중국어 간체 초기 지원 locale을 `zh-CN`으로 정했다. #67은 `templates/locales/{locale}/...` 미러 구조와 `templates/manifest.json`의 `localization` 계약을 추가했지만, 실제 locale pack 본문은 아직 `planned` 상태다.

#68은 #65의 다국어 정책과 #67의 manifest/update 구조를 받아 실제 영어 pack과 한국어 보존 pack을 채우는 첫 본문 작성 작업이다. 후속 #69는 중국어 간체 pack, #70은 적용/update workflow의 locale 선택 연결, #71은 smoke 검증과 migration guide를 담당한다.

## 범위

### 포함

- `templates/locales/en/...` 영어 locale pack 작성
- 기존 한국어 원문을 `templates/locales/ko/...` 기준으로 보존
- manifest의 `localization.enabled: true` entry와 실제 locale source 경로 정합성 확인
- `AGENTS.md`, `CLAUDE.md`, GitHub Issue/PR template, `mydocs/_templates`, 폴더 README의 영어/한국어 pack 정렬
- manual/Skill은 #67의 `requiresSemanticReview: true` 기준에 맞춰 번역 필요성과 포함 범위를 먼저 확정하고, 포함 시 절차 의미 보존 검증을 수행
- `{REPO_SLUG}`, `{BASE_BRANCH}`, `{issue}`, `local/task{N}` 같은 placeholder, branch pattern, filename pattern, command/code block 보존 검증
- 영어/한국어 핵심 용어 대응표 또는 번역 메모 작성

### 제외

- 중국어 간체 `zh-CN` locale pack 작성
- CLI locale 선택 기능 구현 또는 `.hyper-waterfall/version.json` locale 저장 위치 확정
- 릴리스 생성, checksum 확정, migration guide 작성
- 필요성이 확정되지 않은 내부 manual/Skill 전체의 무리한 번역
- #66에서 이미 다룬 README 다국어 진입 문서의 구조 재개편

## 설계 방향

- locale pack 경로는 #67에서 정한 `templates/locales/{locale}/...` 미러 구조를 따른다.
- `en`은 신규 적용 기본 pack이므로 사용자-facing 문서의 자연어 기준이 된다.
- `ko`는 기존 `templates/` 한국어 원문을 보존하는 pack이다. 한국어 고정 정책 문구는 M050 정책에 맞춰 "선택한 Hyper-Waterfall locale을 따른다"는 의미로 바꾸되, 기존 운영 맥락과 절차 의미는 보존한다.
- `templates/` root의 현재 canonical source는 locale source 누락 판단과 release checksum 기준으로 유지한다. 이번 task에서 locale source를 추가하더라도 manifest 구조 자체를 크게 바꾸지 않는다.
- manual/Skill은 에이전트 절차 계약이므로 `requiresSemanticReview: true` 대상이다. 전체 번역이 필요하다고 판단한 파일만 포함하고, 포함/제외 근거를 기술 기록에 남긴다.
- 영어 번역은 OpenAI/Anthropic 특정 제품명에 종속되지 않는 모델 중립 표현을 우선한다. 단, Issue, PR, label, milestone, commit, branch처럼 플랫폼 식별자나 명령어는 번역하지 않는다.
- 번역 중 표현이 흔들리는 용어는 용어 대응표에서 한국어 원문, 영어 기준 표현, 보존 이유를 함께 기록한다.

## 잠정 단계

- **Stage 1 — locale 대상 inventory와 번역 범위 확정**
  - manifest의 `localization.enabled: true` entry, 현재 `templates/` 원문, 한국어 고정 문구, manual/Skill 포함 필요성을 조사한다.
  - 산출물: `mydocs/tech/task_m050_68_translation_terms.md` 초안과 Stage 1 보고서
  - 검증 관점: #65/#67 정책과 #68 포함/제외 범위가 충돌하지 않는지
- **Stage 2 — 영어 기본 locale pack 작성**
  - `templates/locales/en/...`에 기본 pack을 작성한다.
  - 산출물: 영어 AGENTS/CLAUDE, GitHub template, `_templates`, 폴더 README, 필요 시 manual/Skill 영어본
  - 검증 관점: placeholder, 명령어, branch/file pattern이 번역되지 않았는지
- **Stage 3 — 한국어 원문 보존 pack 작성**
  - 기존 한국어 원문을 `templates/locales/ko/...`에 보존하고 M050 언어 정책 문구만 필요한 범위에서 정렬한다.
  - 산출물: 한국어 locale pack과 용어 대응표 보강
  - 검증 관점: 기존 한국어 운영 맥락과 절차 의미가 손실되지 않았는지
- **Stage 4 — pack 정합성 검증과 smoke 기대값 정렬**
  - manifest sourcePattern과 실제 파일 존재 여부, CLI doctor/init smoke 출력, locale별 주요 template diff를 확인한다.
  - 산출물: Stage 4 보고서와 필요한 최소 테스트/문서 보정
  - 검증 관점: 영어 pack이 기본 locale로 사용 가능하고, 한국어 pack이 보존 기준으로 기능하는지

## 리스크

- **절차 의미 drift**: manual/Skill 번역에서 승인 게이트나 issue 기준 추적 의미가 약해질 수 있다. `requiresSemanticReview: true` 대상은 용어 대응표와 수동 diff 검토를 함께 수행한다.
- **범위 과대화**: manual/Skill 전체 번역이 task 규모를 크게 키울 수 있다. Stage 1에서 필수 포함 범위와 제외 근거를 먼저 확정하고, 불필요한 전체 번역은 후속 후보로 남긴다.
- **한국어 원문 손실**: 기존 `templates/` 한국어 문구를 선택 locale 정책으로 바꾸는 과정에서 원래 운영 맥락이 사라질 수 있다. `ko` pack에는 보존 기준을 남기고, root source 수정은 불일치 해소가 필요한 최소 범위로 제한한다.
- **부분 pack 혼합**: 일부 경로만 영어/한국어 pack에 생기면 적용 결과가 혼합 locale이 될 수 있다. Stage 4에서 manifest sourcePattern별 존재 여부를 검증한다.

## 승인 요청 사항

- #68을 위 범위와 단계로 진행한다.
- `en` pack은 신규 적용 기본 pack으로 작성하고, `ko` pack은 기존 한국어 원문 보존 pack으로 만든다.
- manual/Skill은 Stage 1에서 필수 포함 범위를 확정한 뒤 필요한 파일만 번역하고, 포함/제외 근거를 `mydocs/tech/task_m050_68_translation_terms.md`에 남긴다.
- 중국어 간체 pack, CLI locale 선택 구현, migration guide는 각각 #69, #70, #71로 유지한다.

승인되면 `task_m050_68_impl.md`에서 단계별 산출물, 검증 명령, 커밋 메시지를 구체화한다.
