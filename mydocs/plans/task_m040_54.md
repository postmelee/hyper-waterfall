# Task #54 수행계획서 - Claude plugin public marketplace 배포와 release asset 게시

GitHub Issue: [#54](https://github.com/postmelee/hyper-waterfall/issues/54)
마일스톤: M040

## 목적

Claude plugin local/zip 후보를 public marketplace 또는 GitHub Release asset으로 공개할 수 있는지 판단하고, 승인된 범위에서 배포 또는 보류 결과를 확정한다. 결과는 public 설치 경로, 대체 설치 경로, release asset 정책, rollback/보류 조건까지 추적 가능한 문서로 남긴다.

이번 task는 #40에서 smoke를 통과한 `plugins/claude/hyper-waterfall/` 후보를 대상으로 한다. 공개 제출과 release asset 게시는 외부 공개 action이므로, 기술 검증과 작업지시자 별도 승인 게이트를 분리한다.

## 배경

#40은 Claude Code `2.1.143`에서 directory `--plugin-dir` load, zip `--plugin-dir` load, component inventory smoke를 통과했다. 다만 public marketplace 배포와 release asset 게시는 제출 권한, 공개 설명 문구, release asset 정책, rollback 조건이 필요하므로 #54로 분리했다.

#52는 Codex plugin public 배포와 UI discovery smoke를 진행 중이다. #54와 직접 산출물은 다르지만 public 배포 문서, 공통 배포 원칙, #41 최종 채널 감사 인계 항목은 겹칠 수 있다. 따라서 #54는 별도 worktree `../hyper-waterfall-task54`에서 진행하고, 공통 문서 변경 전후로 #52 변경 가능성을 재확인한다.

## 설계 방향

- #54는 `local/task54` 브랜치의 분리 worktree에서 진행한다. 루트 worktree의 미추적 파일과 #52 진행 변경을 건드리지 않는다.
- 공개 action은 승인 게이트를 둘로 나눈다. Stage 1-2는 read-only 조사와 artifact 후보 점검, Stage 3 이후 public marketplace 제출 또는 release asset 게시가 필요하면 작업지시자에게 별도 승인 요청을 올린다.
- Claude plugin 후보는 계속 hook 없는 thin wrapper로 유지한다. public 제출을 위해 metadata와 안내 문구를 보강할 수는 있지만 canonical Skill/manual/manifest를 plugin 내부에서 새로 재작성하지 않는다.
- release asset은 GitHub Release/tag와 `templates/manifest.json` 기준을 흔들지 않는 보조 배포물로만 둔다. asset을 게시한다면 파일명, checksum, 생성 명령, smoke 명령, rollback 조건을 기술 기록에 남긴다.
- #52와의 충돌 가능성이 있는 공통 문서는 늦게 수정하고, 수정 전 `git fetch origin` 및 #52 공개 브랜치/PR 존재 여부를 재확인한다. 공통 문서가 이미 #52에서 바뀌었으면 #54 문서는 Claude-specific 섹션만 좁게 조정한다.

## 리스크

- **#52 공통 문서 충돌**: #52도 public plugin 배포 문서를 수정할 수 있다. #54는 분리 worktree에서 진행하고, 공통 문서 수정 전후로 origin과 #52 상태를 재확인한다.
- **public 제출 권한 불확실성**: marketplace 제출 권한이나 제출 surface가 없으면 public 배포는 NO-GO로 기록하고 대체 설치 경로를 유지한다.
- **release asset 정책 미확정**: v0.2.0 release에 Claude plugin zip을 포함할지 불명확하면 게시하지 않고 checksum과 생성 절차만 후보로 남긴다.
- **canonical drift**: public 안내 문구가 core Skill이나 manual을 요약해 새 절차처럼 만들면 drift가 생긴다. README와 wrapper는 canonical 경로와 fallback 안내 중심으로 제한한다.
- **외부 공개 action rollback 비용**: marketplace 제출, release asset 게시, unpublish/delete는 외부 상태를 바꾸므로 실행 전 승인과 rollback 조건을 명시한다.
- **환경 의존 smoke**: Claude Code version, 로그인 상태, marketplace 접근 권한, 네트워크 상태에 따라 smoke가 제한될 수 있다. 실패와 미수행을 구분해 기록한다.
