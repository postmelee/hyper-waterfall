# Task #22 수행계획서 - README 상단 빠른 적용 안내 추가

GitHub Issue: [#22](https://github.com/postmelee/hyper-waterfall/issues/22)
마일스톤: M020

## 목적

README 첫 화면 근처에서 사용자가 Hyper-Waterfall을 새 저장소에 적용하려면 무엇을 AI에게 요청해야 하는지 빠르게 확인할 수 있게 한다.

기존 `새 저장소에 빠르게 적용하기` 섹션의 상세 안내는 유지하되, README 상단에는 3~5줄 수준의 짧은 진입 안내만 추가해 첫 방문자의 탐색 비용을 낮춘다.

## 배경

현재 README에는 `새 저장소에 빠르게 적용하기` 섹션과 `npx hyper-waterfall` dry-run 예시가 있지만, 첫 화면 상단에서는 사용자가 바로 실행하거나 AI에게 요청할 문구를 찾기 어렵다.

v0.2.0 배포 준비와 함께 README 상단에 빠른 적용 안내를 두면 사용자가 긴 설명을 읽기 전에 적용 경로를 이해할 수 있다. 다만 npm publish, GitHub Release/tag가 확정되기 전 상태를 고려해 `install` 완료처럼 배포 상태를 앞서 암시하는 표현은 피해야 한다.

참고 기준은 `docs/agent-entrypoint.md`, 기존 README의 `새 저장소에 빠르게 적용하기` 섹션, `docs/distribution-channels.md`, `templates/manifest.json`, migration guide다.

## 범위

### 포함

- README 첫 화면 근처에 3~5줄 수준의 빠른 적용 안내 추가
- AI에게 보낼 적용 프롬프트와 `docs/agent-entrypoint.md` 기준의 짧은 연결
- npm CLI 예시를 dry-run 또는 보조 채널로만 표현하도록 문구 점검
- 기존 `새 저장소에 빠르게 적용하기` 섹션과 역할 중복 여부 점검
- 문서 grep, `git diff --check`, README diff 수동 확인

### 제외

- npm publish 실행
- GitHub Release/tag 생성
- Homebrew, Docker, plugin 구현
- CLI 동작 변경
- 매뉴얼 또는 SKILL 본문 변경

## 설계 방향

- 빠른 안내는 README 첫 제목과 프로젝트 설명 직후, `적용하면 바로 달라지는 것` 섹션 이전을 우선 후보로 둔다.
- 상단 안내는 긴 설명을 반복하지 않고, 적용 프롬프트와 기준 문서 링크만 제공한다.
- 확정 배포를 암시하는 `설치 완료`, `바로 install` 중심 표현은 피하고, AI 적용 프롬프트와 dry-run 성격을 분리한다.
- canonical 기준은 GitHub Release/tag, `templates/manifest.json`, migration guide, `docs/agent-entrypoint.md`에 있음을 흐리지 않는다.
- 기존 README 하단 섹션은 상세 안내 역할을 유지하고, 상단 안내와 충돌하는 경우 최소 수정으로 정합성을 맞춘다.

- **Stage 3 - 정합성 검증과 보고**
  - 이슈 검증 명령을 실행하고 README diff를 수동 확인한다.

### 통합 검증

- README 첫 화면 근처에서 사용자가 적용 시작 방법을 빠르게 확인할 수 있다.
- 안내 문구가 실제 배포 상태보다 앞서 `install` 완료를 암시하지 않는다.
- `docs/agent-entrypoint.md`가 적용 절차 기준이라는 점이 유지된다.
- 기존 `새 저장소에 빠르게 적용하기` 섹션과 역할이 중복되지 않는다.
- canonical 기준인 GitHub Release/tag, `templates/manifest.json`, migration guide를 흐리지 않는다.
- `git status --short`가 PR 준비 전 빈 출력이다.
- `git diff --check`가 경고 없이 통과한다.

## 리스크

- **상단 안내와 기존 섹션 중복**: 첫 화면 안내는 요약 역할만 하도록 제한하고, 상세 설명은 기존 섹션에 둔다.
- **배포 상태 오인**: npm CLI와 GitHub Release/tag가 확정 배포처럼 보이지 않도록 dry-run과 보조 채널 표현을 유지한다.
- **canonical 기준 약화**: 적용 기준 문구는 `docs/agent-entrypoint.md`, GitHub Release/tag, manifest, migration guide와 연결한다.
