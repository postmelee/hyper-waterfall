# 오늘할일 템플릿

이 파일은 `mydocs/orders/{yyyymmdd}.md` 작성용 중앙 템플릿이다. 실제 오늘할일 파일은 `orders/` 폴더에 날짜 파일명으로 만들고, 이 템플릿 파일 자체를 `orders/`에 복사해 보관하지 않는다.

## 사용 위치

- 실제 파일: `mydocs/orders/{yyyymmdd}.md`
- 작성 시점: 타스크 시작, 단계 경계에서 상태가 바뀔 때, 최종 보고와 merge 후 정리 시점
- 작성 언어: 한국어

## 본문 템플릿

```md
# 오늘 할일 - YYYY년 M월 D일

## M{마일스톤} — {마일스톤 이름}

| Issue | 타스크 | 상태 | 비고 |
|------|--------|------|------|
| #{번호} | {작업 한 줄 요약} | 예정 | {메모} |
| #{번호} | {작업 한 줄 요약} | 진행중 | {메모} |
| #{번호} | {작업 한 줄 요약} | 완료 | 완료: HH:mm, {완료 요약} |
```

## 공통 운영 섹션 템플릿

특정 마일스톤에 속하지 않는 운영 작업만 사용한다.

```md
## 공통 — 운영 작업

| Issue | 타스크 | 상태 | 비고 |
|------|--------|------|------|
| 생성필요 | {작업 한 줄 요약} | 예정 | {메모} |
```

## 백로그 템플릿

오늘의 주 작업은 아니지만 잊지 말아야 할 항목이 있을 때만 사용한다.

```md
## 백로그

| No | 내용 | 발견일 | 상태 | 비고 |
|----|------|--------|------|------|
| B-001 | {항목} | YYYY-MM-DD | 미해결 | {메모} |
```

## 상태값

- `예정`
- `진행중`
- `완료`
- `보류`

## 작성 규칙

- GitHub Issue 번호가 있으면 `#번호` 형식으로 적는다.
- 아직 이슈가 없으면 임의 번호나 `미정` 대신 `생성필요`로 적는다.
- 완료 작업은 비고에 `완료: HH:mm`을 포함한다.
- 상세 분석, 검증 로그, 기술 조사 내용은 `orders/`에 쓰지 않고 적절한 문서 폴더로 분리한다.
