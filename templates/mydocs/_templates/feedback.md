# 피드백 템플릿

이 파일은 `mydocs/feedback/`에 작업지시자 피드백, 코드 리뷰 의견, 재작업 지시를 기록할 때 사용하는 중앙 템플릿이다. 피드백 문서는 AI가 다음 수정에서 놓치지 말아야 할 판단과 요청을 보존하는 용도이며, 장기 기술 조사나 트러블슈팅 기록을 대신하지 않는다.

## 사용 위치

- 실제 파일: `mydocs/feedback/{yyyymmdd}_{topic}.md` 또는 `mydocs/feedback/task_{milestone}_{issue}_feedback.md`
- 작성 시점: 단계 승인 보류, PR 리뷰, 작업지시자 재작업 요청, 설계 방향 수정 요청 시점
- 작성 언어: 한국어

## 대상

- 관련 이슈: #{issue}
- 관련 Stage: {stage 또는 해당 없음}
- 관련 PR: #{pr 또는 해당 없음}
- 작성자: {작업지시자 또는 리뷰어}
- 작성일: YYYY-MM-DD

## 피드백 요약

{수정 요청의 핵심을 1~3문장으로 적는다.}

## 상세 피드백

| 위치 | 내용 | 기대 결과 |
|---|---|---|
| `{file 또는 문서 섹션}` | {문제 또는 요청} | {어떻게 바뀌어야 하는지} |

## 우선순위

- must-fix: {반드시 반영해야 할 사항}
- should-fix: {가능하면 반영할 사항}
- note: {참고 사항}

- {검증 명령 또는 확인 방법}
