# `tech/` 文件夹规则

## 目的

把技术研究、结构分析、规格审查和备选方案比较作为可复用的长期证据保留下来。

## 回答的问题

- “发现了什么？”
- “为什么选择这个设计？”

## 何时编写

实施前研究、设计备选方案比较、外部文档/规格审查、结构分析或非正式草稿编写之前。

## 允许的文件名

- `{yyyymmdd}_{topic}.md`
- `task_{milestone}_{issue_number}_{topic}.md`

## 使用的模板

`mydocs/_templates/tech_note.md`

## 必需内容

- 研究背景
- 研究问题
- 研究对象
- 发现
- 决策
- 非决策或延后事项
- 实施影响
- 参考资料

## 不允许的内容

- 每日 task 状态表
- Stage 完成审批请求
- 以防复发为重点的失败记录
- 应属于官方文档根的用户、贡献者或 API 合约文档

## 下一次会话 AI 必须恢复的上下文

已研究的事实、不应重复收集的证据，以及未解决的问题。
