# `_templates/` 文件夹规则

## 目的

在一个中央位置定义 Hyper-Waterfall 产出物的输出格式。

## 回答的问题

“此文档应遵循哪些章节和约束？”

## 何时编写

当新增一种产出物类型，或现有产出物输出格式发生变化时。

## 允许的文件名

- `orders.md`
- `task_plan.md`
- `task_impl_plan.md`
- `stage_report.md`
- `final_report.md`
- `feedback.md`
- `tech_note.md`
- `troubleshooting.md`
- `external_pr_review.md`
- `external_pr_review_impl.md`
- `external_pr_report.md`
- 其他能够清楚标识产出物类型的名称

## 使用的模板

不适用。此文件夹本身就是模板的真相来源。

## 必需内容

- 实际文件位置
- 何时编写
- 编写语言
- 必需章节
- 可选章节
- 验证或审批标准

## 不允许的内容

- 某个具体 task 的实际验证日志
- 完成报告
- 任务请求者审批记录

## 下一次会话 AI 必须恢复的上下文

每种产出物类型的输出格式，以及每份文档所属的文件夹。
