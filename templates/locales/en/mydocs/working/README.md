# `working/` Folder Rules

## Purpose

Record how each Stage ended and request approval to enter the next Stage.

## Questions Answered

- "How far did the work get?"
- "Did verification pass?"
- "What impact does this have on the next Stage?"

## When to Write

Immediately after one Stage is implemented and verified, before moving to the next Stage.

## Allowed Filename

`task_{milestone}_{issue_number}_stage{N}.md`

## Template Used

`mydocs/_templates/stage_report.md`

## Required Content

- Stage purpose
- Artifacts
- Body change scope or lossless preservation
- Verification results
- Residual risks
- Impact on next Stage
- Approval request

## Content Not Allowed

- Final reports
- Completion declarations for a Stage whose verification is still failing
- Unauthorized changes to the next Stage implementation plan

## Context the Next Session AI Must Restore

The last approved Stage, remaining risks, and conditions to carry into the next Stage.
