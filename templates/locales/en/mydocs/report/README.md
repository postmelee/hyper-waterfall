# `report/` Folder Rules

## Purpose

Close and preserve the full plan-implementation-verification cycle for a task.

## Questions Answered

- "What was the result?"
- "Were the acceptance criteria satisfied?"
- "What remains?"

## When to Write

After all Stages and integrated verification are complete, before PR publication.

## Allowed Filename

`task_{milestone}_{issue_number}_report.md`

## Template Used

`mydocs/_templates/final_report.md`

## Required Content

- Work summary
- Changed file list and impact area
- Before/after comparison
- Verification results
- Residual risks and follow-up work
- Approval request

## Content Not Allowed

- In-progress Stage reports
- Completion declarations for unverified results
- Documents that contain only the short PR review body

## Context the Next Session AI Must Restore

The final task state, pre-merge verification evidence, and follow-up candidates.
