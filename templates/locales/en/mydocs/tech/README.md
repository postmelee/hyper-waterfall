# `tech/` Folder Rules

## Purpose

Preserve technical research, structural analysis, specification review, and alternative comparisons as reusable long-term evidence.

## Questions Answered

- "What was discovered?"
- "Why was this design selected?"

## When to Write

Before implementation research, design alternative comparison, external document/spec review, structural analysis, or unofficial draft writing.

## Allowed Filenames

- `{yyyymmdd}_{topic}.md`
- `task_{milestone}_{issue_number}_{topic}.md`

## Template Used

`mydocs/_templates/tech_note.md`

## Required Content

- Research background
- Research questions
- Research targets
- Findings
- Decisions
- Non-decisions or deferred items
- Implementation impact
- References

## Content Not Allowed

- Daily task status tables
- Stage completion approval requests
- Failure records focused on recurrence prevention
- User, contributor, or API contract documents that belong in the official documentation root

## Context the Next Session AI Must Restore

Facts already researched, evidence that should not be re-collected, and unresolved questions.
